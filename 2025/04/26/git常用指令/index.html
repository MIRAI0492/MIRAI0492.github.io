规范

分支

  1. main/master：生产环境代码
  2. develop：开发分支
  3. feat/xxx：功能分支

提交

git commit -m “fix: resolve login timeout issue”

开发流程

  1. git checkout -b my-feature # 本地新建分支
  2. 开发并提交代码
  3. git push origin my-feature # 推送到远程
  4. 在GitHub/GitLab创建Merge Request
  5. 代码审查后合并
